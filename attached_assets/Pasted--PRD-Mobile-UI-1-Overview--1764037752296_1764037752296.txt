📘 웹 주문·관리 시스템 PRD
(Mobile UI 최적화 버전)
1. 제품 개요 (Overview)

본 프로젝트는 매장에서 사용할 모바일 웹 기반 주문 관리 시스템이다.
운영자는 메뉴 생성/관리부터 주문 입력 및 제공 확인, 결제 처리까지 모든 업무를 스마트폰으로 처리할 수 있다.

모바일 환경을 고려하여 큰 버튼, 단순한 구조, 적은 입력, 스크롤 최소화, 명확한 인터랙션을 목표로 한다.

2. 주요 페이지 구성 (Main Pages)

메인 페이지(index.html)에서 아래 5개 기능으로 이동:

관리

주문

주문확인

음식확인

계산

모바일 UI 최적화 기준
✔︎ 버튼은 최소 높이 60~70px
✔︎ 버튼 간 간격 12~16px
✔︎ 폰트 크기 16px 이상
✔︎ 탭/드래그 대신 단순 클릭으로 조작
✔︎ 리스트는 세로 스크롤 기준 레이아웃
✔︎ 테이블 구조보다 카드형 UI 우선

3. 기능 상세 요구사항
3.1 메인 페이지 (index.html)
● UI 구성

화면 전체 폭을 활용한 5개의 대형 버튼

관리

주문

주문확인

음식확인

계산

● UX 요구사항

세로 스크롤 없음

버튼은 카드형 또는 네모 박스 형태

아이콘 포함 가능 (예: 주문=🧾, 계산=💰)

3.2 관리 페이지 (메뉴 관리)
● 목적

메뉴명/가격 신규 등록, 수정, 삭제, 판매 마감 처리.

● 화면 구성

메뉴 등록 섹션

메뉴명 입력 (텍스트 입력창)

가격 입력 (숫자 입력창)

저장 버튼(대형 버튼)

메뉴 리스트

카드 형태 UI

메뉴명 (bold)

가격

버튼 3개:

수정

삭제

마감 버튼 (sale_yn = 'N'으로 변경)

● 기능 요구사항

새 메뉴 저장 시

sale_yn = 'Y'

마감 시 주문 페이지에 노출되지 않음

모바일에서 수정/삭제 버튼은 터치 간격 확보

3.3 주문 페이지
● 목적

고객 성함을 입력하고 메뉴 수량을 선택하여 주문 생성.

● 화면 구성
1) 성함 입력 화면

간단한 입력 필드

“확인” 버튼 클릭 → 다음 단계로 이동

2) 메뉴 선택 화면

판매중(sale_yn = Y) 메뉴만 카드 형태로 나열

카드는 아래 구조로 구성

메뉴명

가격

수량 조절 버튼

“–”

현재 수량

“+”

하단 고정 버튼: 주문등록

● 기능 요구사항

수량 기본값 = 0

/ – 버튼은 모바일에서 조작이 쉬운 크기로 설계

주문등록 시 수량 > 0인 메뉴만 저장

ORDER 테이블 저장값:

name

menu_name

price

qty

pay_yn = 'N'

serve_yn = 'N'

use_yn = 'Y'

3.4 주문확인 페이지 (고객별 주문)
● 목적

주문을 성함별로 조회하여 제공 완료 또는 주문 취소를 처리.

● 화면 구성
1) 성함 리스트

최근 주문이 상단에 오도록 정렬

카드 형태로 나열

2) 성함 선택 시 주문 상세 표시

메뉴명

가격

수량

제공여부 표시 (Y/N)

3) 기능 버튼

제공완료 (serve_yn = 'Y')

주문취소 (use_yn = 'N')

● 모바일 UX 요구사항

화면 전환 최소화

상세 주문은 아코디언 방식 or 모달 방식 가능

주문취소는 실수 방지를 위해 “정말 취소할까요?” 팝업 필요

3.5 음식확인 페이지 (메뉴별 제공 체크)
● 목적

메뉴별로 주문된 건들을 확인하고 제공여부를 수정 가능.

● 화면 구성
1) 메뉴 리스트

메뉴명만 나열한 카드 리스트

2) 메뉴 선택 시 주문 리스트 출력

주문 내용 카드

성함

수량

제공여부 체크박스

하단에 저장 버튼

● 기능 요구사항

체크박스 변경 후 저장 시
해당 ORDER 레코드들의 serve_yn 값 변경

use_yn = 'Y'인 주문만 표시

3.6 계산 페이지 (결제 처리)
● 목적

미결제 고객을 조회하고 결제 완료 처리.

● 화면 구성
1) 성함 리스트

pay_yn = N, use_yn = Y인 성함만 표시

2) 성함 클릭 시 주문 상세 표시

메뉴명

가격

수량

소계(가격×수량)

하단에 총 합계 금액 표시

3) 기능 버튼

계산완료 (pay_yn = 'Y')

● 모바일 UX 요구사항

총 금액은 고정 하단바에서 표시

결제완료 버튼은 큰 크기로

4. 데이터베이스 요구사항
4.1 MENU 테이블
컬럼명	타입	설명
id	PK	메뉴 ID
menu_name	VARCHAR	메뉴명
price	INT	가격
sale_yn	CHAR(1)	판매 여부(Y/N)
4.2 ORDER 테이블
컬럼명	타입	설명
id	PK	주문 ID
name	VARCHAR	성함
menu_name	VARCHAR	메뉴명
price	INT	가격
qty	INT	수량
serve_yn	CHAR(1)	제공여부 (Y/N)
pay_yn	CHAR(1)	계산여부 (Y/N)
use_yn	CHAR(1)	사용여부 (Y/N → 취소 시 N)
created_at	DATETIME	주문 시간
5. 예외 처리 요구사항

가격은 숫자만 입력 가능

수량은 음수 불가

판매중(sale_yn = 'Y') 메뉴만 주문 가능

주문취소(use_yn = 'N') 발생 시

계산/제공/조회 페이지 모두에서 제외

계산완료 후에는 수정 불가

제공완료 후에도 계산은 별개 처리

6. 모바일 UX 설계 기준
✔ 버튼 영역 최소 60px
✔ 폰트 사이즈 최소 16px
✔ 터치 영역 여백 충분히 확보
✔ 색 대비 높게 디자인 (가독성 강화)
✔ 입력 최소화 (자동 포커스, 숫자키패드 활성화)
✔ 리스트는 카드형 구성
✔ 스크롤 깊이 최소화
