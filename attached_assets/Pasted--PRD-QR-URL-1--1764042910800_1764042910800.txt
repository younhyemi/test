📘 모바일 웹 주문 시스템 — 최종 PRD
(관리자 모드 + 고객 모드 + 테이블번호 기반 + QR URL 지원)
1. 프로젝트 목적

본 시스템은 매장에서 스마트폰을 통해 주문을 받고, 주방과 관리자가 실시간으로 주문 현황을 확인하고 결제까지 처리할 수 있는 모바일 웹 주문 관리 시스템이다.

테이블 번호 또는 QR 스캔으로 간단하게 접속

고객은 자신의 테이블에 대한 기능만 접근 가능

관리자는 메뉴/주문/음식/계산 전체를 관리 가능

모든 페이지는 모바일 UI 최적화

2. 전체 시스템 구성도

메인 페이지(index.html) → 테이블번호 입력

입력값이 "admin" → 관리자 모드

그 외 "숫자 또는 문자열" → 고객 모드

QR 코드 스캔 시 자동으로 ?table=번호 형태로 해당 테이블 페이지로 이동 가능

3. 페이지 상세 요구사항
3.1 메인 페이지 (index.html)
● 기능

테이블번호 입력 필드

"입장하기" 버튼

● 로직

입력값 "admin"이면 → 관리자 페이지로 이동

그 외는 → 고객 페이지로 이동

QR 접속 시 index.html?table=5 형태로 자동 테이블번호 입력 상태로 열림

● 모바일 UI

화면 중앙 정렬

입력 필드 큰 크기 (height 50px 이상)

버튼 height 60px 이상

배경에 간단한 테이블 이미지 or 브랜드 컬러 사용 가능

4. 관리자 페이지

관리자 화면에는 4개의 버튼이 표시된다.

[메뉴관리]
[테이블별 주문확인]
[음식확인]
[계산]


다음은 각 기능의 상세 설명이다.

4.1 1) 메뉴관리
● 목적

메뉴명/가격 관리, 판매 가능 여부 설정

● 화면 구성

메뉴명 입력

가격 입력

"저장" 버튼

메뉴 리스트 (카드형 UI)

메뉴명

가격

"수정"

"삭제"

"마감" 버튼

● 기능 상세
✔ 메뉴 저장

MENU 테이블에 저장

기본값:

sale_yn = 'Y'

✔ 메뉴 수정

항목 선택 후 수정 가능

✔ 메뉴 삭제

DB에서 삭제

✔ 마감 처리

sale_yn = 'N'으로 변경

주문 페이지에서 해당 메뉴가 노출되지 않음

✔ 모든 이벤트 후 자동 새로고침
4.2 2) 테이블별 주문확인
● 목적

테이블 단위 주문을 확인하고, 제공완료/주문취소 처리

● 화면 구성

주문을 등록한 테이블번호 리스트

테이블번호 클릭 시 해당 주문 리스트 표시

메뉴명

수량

가격

제공여부(Y/N)

버튼 2개

제공완료 (serve_yn = 'Y')

주문취소 (use_yn = 'N')

● 기능 상세

제공완료 클릭 시:

해당 테이블 주문 전체 serve_yn = 'Y'

주문취소 클릭 시:

use_yn = 'N' (모든 페이지에서 비노출)

✔ 버튼 클릭 후 자동 새로고침
4.3 3) 음식확인
● 목적

메뉴 기준으로 주문 현황을 보고 제공 여부를 체크로 수정 가능

● 화면 구성

MENU 메뉴명 리스트

메뉴 클릭 시 해당 메뉴 주문 리스트

메뉴명

테이블번호

수량

완료여부 체크박스 (serve_yn Y/N)

"저장" 버튼

● 기능 상세

체크/체크해제 모두 가능

저장 시 해당 ORDER 레코드의 serve_yn 값을 업데이트

실수 방지를 위해 체크한 항목도 다시 해제 가능

✔ 저장 후 자동 새로고침

미결제 테이블을 확인하고 결제를 완료 처리

● 화면 구성

테이블번호 클릭 → 주문 상세 리스트 표시

메뉴명

가격

수량

소계

하단: 총 합계

pay_yn = 'Y'

✔ 계산완료 후 자동 새로고침
5. 고객 페이지

고객 페이지는 테이블번호 입력 or QR 접속을 통해 입장.

● 기능 버튼
[주문]
[주문확인]


QR 접속 시 URL 예시:

https://사이트주소/order.html?table=5


→ 테이블번호 자동 세팅

5.1 1) 주문
● 목적

고객이 메뉴 선택 후 주문 등록

● 화면 구성

메뉴 리스트 (판매중인 메뉴만):

메뉴명

가격

수량조절 버튼 (+ / –)

하단: "주문등록"

● 기능 상세

수량 기본값 = 0

/ – 클릭로 증감

주문등록 시 qty > 0인 메뉴만 저장

저장 값:

테이블번호(TABLE_NO)

메뉴명

가격

qty

pay_yn = 'N'

serve_yn = 'N'

use_yn = 'Y'

● 목적

현재 테이블에서 주문한 내용을 고객이 확인

● 화면 구성

수량

가격

소계

6. 데이터베이스 모델
6.1 MENU 테이블
컬럼명	타입	설명
id	PK	메뉴 ID
menu_name	VARCHAR	메뉴명
price	INT	가격
sale_yn	CHAR(1)	판매여부 (Y/N)
6.2 ORDER 테이블
컬럼명	타입	설명
id	PK	주문 ID
table_no	VARCHAR	테이블번호
menu_name	VARCHAR	메뉴명
price	INT	단가
qty	INT	주문수량
serve_yn	CHAR(1)	제공여부(Y/N)
pay_yn	CHAR(1)	계산여부(Y/N)
use_yn	CHAR(1)	사용여부(Y/N → 주문취소 시 N)
created_at	DATETIME	주문시간
7. 모바일 UI 요구사항

모든 버튼 높이 ≥ 60px

카드형 UI 중심

입력 필드 큰 폰트(16px+)

스크롤 최소화

목록은 세로 정렬

버튼 간 간격 12~16px

중요 버튼은 하단 고정 가능

“실수 방지용” 확인 팝업 필요 (취소, 계산 등)
