웹사이트를 개발하고 싶어.
메인페이지는 "index.html"이야.
해당 페이지는 모바일에서 사용할 예정이라 모바일 UI에 최적화해줘.

메인페이지에서 테이블번호를 입력해달라는 칸이 보여.

1. 테이블번호에 "admin" 입력 시 관리자 페이지로 이동
관리자페이지에는 4개의 버튼이 보여.
"메뉴관리", "테이블별 주문확인", "음식확인", "계산"
 
1-1) "메뉴관리" 버튼을 누르면
메뉴를 관리할 수 있는 페이지로 이동해.
구분명과 메뉴명과 가격을 입력할 수 있고 수정 및 삭제도 가능해.
구분명과 메뉴명과 가격을 입력 후 저장하게되면
데이터베이스 MENU 테이블에 구분명, 메뉴명, 가격이 저장이돼.
메뉴명과 가격 데이터들에 각각 "마감"이라는 버튼이 있고
해당 버튼을 누르게 되면 손님이 주문 할 수 없어.
조회될 때는 구분명, 메뉴명, 가격 순으로 조회 돼.
모든 버튼 클릭 이벤트 처리 후에는 자동으로 새로고침돼.

1-2) "테이블별 주문확인" 버튼을 누르면
주문(ORDER)을 등록한 테이블번호들이 리스트업되고
테이블번호을 누르면 해당 테이블번호으로 주문한 메뉴들의 메뉴명, 수량, 가격, 주문등록일시, 제공일시, 계산여부와 계산을 했다면 계산일시가 리스트업해서 보여져.
각 행에는 제공완료버튼 / 제공취소버튼 / 주문취소버튼이 보여지고
하단에는 전체 제공완료를 할 수 있는 제공완료 버튼과 전체 제공취소를 할 수 있는 제공취소버튼과 전체 주문취소를 할 수 있는 주문취소 버튼이 있어

제공완료 버튼을 누르면 ORDER 테이블에 제공여부 컬럼이 "Y"로 변경돼.
제공취소 버튼을 누르면 ORDER 테이블에 제공여부 컬럼이 "N"로 변경돼.
주문취소 버튼을 누르면 사용여부 컬럼이 "N"으로 변경돼.
제공완료 or 주문취소 버튼 클릭 이벤트 처리 후에는 자동으로 새로고침돼.

1-3) "음식확인" 버튼을 누르면
계산여부 컬럼이 "Y"이면서 제공여부 컬럼이 "N"인 ORDER 테이블들의 메뉴명들이 리스트업 되어서 보여져.
메뉴명들이 보여질 때 제공해야하는 수량 합계도 함께 보여져.
메뉴명을 누르면
해당 메뉴의 주문(ORDER)이 등록된 메뉴명, 테이블번호, 수량, 주문등록일시가 주문등록일시 순으로 표시되고
각 행마다 제공완료 버튼이 있어서 제공완료(제공여부 "Y") 처리할 수도 있어.
제공완료 버튼 클릭 이벤트 처리 후에는 자동으로 새로고침돼.

1-4) "계산" 버튼을 누르면
주문(ORDER)을 등록한 테이블번호와 수량합계와 가격 금액 합계가 테이블번호 순으로 리스트업되고 
테이블번호를 누르면 해당 테이블번호으로 주문한 메뉴들이 리스트업해서 보여져.
각 행에는 계산완료버튼과 계산취소 버튼이 있고, 계산완료된 건은 계산완료 시간도 함께 보여져.
하단에는 합계가 있어서 전체 총 가격과, 계산완료한 총 가격(계산여부 Y)과 계산해야하는 총 가격(계산여부 N)이 함께 보여져.
테이블번호별 계산 안된 건(계산여부 N)들을 전체 계산완료(계산여부 Y) 처리 할 수 있는 버튼과 전체 계산취소(계산여부 N)를 할 수 있는 버튼이 하단에 함께 있어.
"계산완료" 버튼을 누르면 데이터베이스 ORDER 테이블의 "계산여부"라는 컬럼에 "Y"로 값이 변경되고 계산완료 시간이 sysdate로 update돼.
"계산취소" 버튼을 누르면 데이터베이스 ORDER 테이블의 "계산여부"라는 컬럼에 "N"로 값이 변경돼.
계산완료 or 계산취소 버튼 클릭 이벤트 처리 후에는 자동으로 새로고침돼.

2. 테이블번호에 "admin"이 아닌 문구 입력 시 고객 페이지로 이동
고객페이지에서는 5개의 버튼이 보여.
"주문", "주문확인"
그리고 QR을 해당 테이블과 연계된 사이트를 카메라로 스캔 시
해당 테이블의 주문과 주문확인이 뜰 수 있었으면 좋겠어.
그러니까 각 테이블번호별로 URL이 달랐으면 좋겠어.

2-1) "주문" 버튼을 누르면
주문을 등록할 수 있는 페이지로 이동해.
 MENU 테이블에 저장되어있는 메뉴명, 가격이 구분명, 메뉴명 순으로 리스트업해서 보여지고
"+" 버튼을 누르면 수량이 늘어나고, "-" 버튼을 누르면 수량이 줄어들어.
수량을 0 이상 입력 한 후 "주문등록" 버튼을 누르면
수량을 0 이상 입력한 메뉴들이 메뉴명/가격/수량/주문등록일시(sysdate) 데이터들로 데이터베이스 ORDER 테이블에 저장이 돼.
ORDER 테이블에 저장될 때는, 처음에 입력한 테이블번호(TABLE_NO)도 함께 저장돼.
ORDER 테이블에는 계산여부와 제공여부 라는 컬럼도 있는데, 주문등록 시 계산여부와 제공여부 컬럼 기본값은 "N"으로 저장돼.
사용여부 라는 컬럼은 기본값으로 "Y"로 저장돼.

2-2) "주문확인" 버튼을 누르면
처음에 입력했던 테이블번호(TABLE_NO)로 등록했던 주문 내역이 보여져.
구분명, 메뉴명, 수량, 가격, 제공일시, 계산일시가 구분명, 메뉴명 순으로 리스트업 되고
제공일시는 제공여부가 Y인 경우만 보이고
계산일시는 계산여부가 Y인 경우만 보여져.
마지막 합계에 전체 합계 가격 금액이 보여져.

이 내용을 정리해서 PRD로 작성해줘.
